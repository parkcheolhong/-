## 🧭 발표용 타임라인 시각화 구조

### 🔹 1. 구성 방식: 기능 흐름 + 반영 상태 + 날짜

```plaintext
[2025-09-28] ─ 표현 카드 컬러 개선 요청 (디자인팀) ─ 🟢 반영 완료
       │
[2025-09-30] ─ 챌린지 XP 기준 명확화 제안 (기획팀) ─ 🟡 진행 중
       │
[2025-10-01] ─ 관리자 대시보드 그래프 변경 요청 (디자인팀) ─ 🟢 반영 완료
       │
[2025-10-02] ─ 지도 오류 처리 로직 개선 요청 (개발팀) ─ 🧪 테스트 중
       │
[2025-10-03] ─ 인터뷰 GPT 톤 조절 옵션 검토 (기획팀) ─ 🔴 미반영
```

> 각 항목은 기능명 + 요청 내용 + 팀 + 상태 + 날짜로 구성  
> 슬라이드에서는 수직 또는 수평 흐름으로 시각화 가능

---

### 🔹 2. 디자인 요소 제안

| 요소 | 구성 방식 |
|------|------------|
| 아이콘 | 🟢 완료 / 🟡 진행 중 / 🔴 미반영 / 🧪 테스트 중  
| 색상 | 상태별 컬러 태그 (초록/노랑/빨강/보라)  
| 폰트 | 제목: Bold / 내용: Regular / 날짜: Light  
| 레이아웃 | 수직 흐름 (세로 타임라인) 또는 수평 흐름 (가로 진행)  
| 강조 | 반영 완료 항목은 배경 강조 또는 체크 아이콘 추가  
| 배경 | 브랜드 컬러 또는 감정별 테마 컬러 적용 가능

### 🔹 4. 확장 방식

- 기능별로 슬라이드 분리 → 표현 카드 / 챌린지 / 지도 / 인터뷰  
- 팀별 요청 흐름 → 디자인팀 / 기획팀 / 개발팀 / 운영팀  
- 반영 상태별 요약 → 완료 / 진행 중 / 테스트 / 미반영  
- 발표 마지막에 “전체 반영률” 또는 “실행력 지표” 시각화 가능
