## 🛠️ GitHub 저장소 생성 및 배포 흐름

### 1️⃣ GitHub 저장소 생성

1. [GitHub](https://github.com) 접속 후 로그인  
2. 우측 상단 “+” 클릭 → “New repository” 선택  
3. 저장소 이름 입력: `nadotong-preview`  
4. 설명 입력: “나도 통역사 Web 프리뷰 데모”  
5. 공개 여부: `Public` 선택  
6. README 파일 생성 체크  
7. “Create repository” 클릭

---

### 2️⃣ 파일 업로드 및 구조 구성

```bash
nadotong-preview/
├── index.html
├── style.css
├── script.js
├── assets/
│   ├── images/
│   └── icons/
├── components/
│   ├── emotionCards.html
│   ├── challenge.html
│   ├── map.html
│   ├── interview.html
│   └── feed.html
```

- GitHub 웹 인터페이스 또는 GitHub Desktop을 통해 파일 업로드  
- 각 기능별 화면을 `components/` 폴더에 분리  
- 이미지와 아이콘은 `assets/` 폴더에 저장

---

### 3️⃣ GitHub Pages 배포 설정

1. 저장소 메인 페이지 → “Settings” 클릭  
2. 좌측 메뉴에서 “Pages” 선택  
3. “Source”에서 `main` 브랜치 + `/root` 선택  
4. “Save” 클릭  
5. 배포 완료 후 링크 생성:  
   `https://housing.github.io/nadotong-preview/`

---

### 4️⃣ Web 프리뷰 링크 공유

- 교육기관, 콘텐츠 기업, SNS 채널에 배포 가능
