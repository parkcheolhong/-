## 🎯 발표용 키비주얼 디자인 시안

### 🔹 1. 전체 레이아웃 구조

```
┌────────────────────────────────────────────┐
│       사용자 테스트 피드백 요약           │
├────────────────────────────────────────────┤
│ 🟦 표현 카드      🟨 챌린지       🟩 지도 표현 │
│ ✅ 진행 중        ✅ 완료         🧪 테스트 중 │
│ “표현 직관적”   “성취감 있음”  “시각화 흥미로움” │
│ 개선: 카드 간격 / 녹음 오류 / 로딩 지연     │
├────────────────────────────────────────────┤
│ 🟥 인터뷰 기능                                │
│ 🔴 미반영                                    │
│ “AI 피드백 공감됨” / 개선: 음성 인식 정확도 │
└────────────────────────────────────────────┘
```

- 상단: 슬라이드 제목 + 브랜드 로고  
- 중단: 기능별 카드형 요약 (색상 태그 + 상태 이모지)  
- 하단: 반영률 요약 + 실행력 강조 메시지

---

### 🔹 2. 카드형 구성 요소

| 요소 | 설명 |
|------|------|
| 기능명 | 표현 카드 / 챌린지 / 지도 / 인터뷰  
| 상태 | ✅ 완료 / 🛠️ 진행 중 / 🧪 테스트 중 / 🔴 미반영  
| 사용자 반응 | 실제 피드백 인용 (예: “표현이 직관적”)  
| 개선 제안 | 기능별 수정 요청 요약  
| 반영 팀 | 디자인팀 / 개발팀 / 기획팀 등

---

### 🔹 3. 시각적 디자인 요소

- **색상 태그**: 기능별 구분 (블루/옐로우/그린/레드)  
- **이모지 강조**: 상태별 시각적 인식 강화  
- **폰트**: Pretendard Bold / 24pt 제목 / 16pt 본문  
- **아이콘**: 기능별 아이콘 (🎨 표현 / 🧠 챌린지 / 🗺️ 지도 / 🎤 인터뷰)  
- **배경**: 밝은 톤 (#F9FAFB) 또는 감정별 그라데이션

---

### 🔹 4. 슬라이드 하단 요약

```
✅ 완료: 1건  
🛠️ 진행 중: 1건  
🧪 테스트 중: 1건  
🔴 미반영: 1건  
총 피드백 항목: 4건  
반영률: 75%
```

> 이 영역은 발표 마무리 시 “실행력”을 강조하는 메시지로 활용  
> 예: “우리는 감정을 중심으로 기능을 설계했고, 75% 이상 반영했습니다.”

## 🧠 활용 팁

- 발표 시 각 카드 클릭 또는 확대 → 기능별 흐름 설명
