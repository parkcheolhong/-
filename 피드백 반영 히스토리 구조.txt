## 🧾 피드백 반영 히스토리 구조

### 🔹 1. 데이터베이스 구조 제안

| 항목 | 설명 |
|------|------|
| 기능명 | 예: 표현 카드 / 챌린지 / 지도 / 인터뷰 / 관리자 대시보드  
| 피드백 내용 | 팀별 피드백 요약 (디자인/기획/개발/운영)  
| 반영 상태 | 🟢 반영 완료 / 🟡 진행 중 / 🔴 미반영 / 🧪 테스트 중  
| 반영 방식 | 수정 / 추가 / 제거 / 보류 / 대체 제안  
| 담당자 | 반영을 수행한 팀원 또는 팀  
| 반영일 | 실제 반영된 날짜  
| 관련 링크 | Figma / GitHub / Web 프리뷰 / Notion 카드  
| 코멘트 | 반영 후 결과 또는 추가 의견

### 🔹 3. 필터/정렬 기능

- 필터: `상태 = 진행 중`, `담당자 = 개발팀`, `기능명 = 챌린지`  
- 정렬: `반영일 역순`, `기능명 오름차순`  
- 그룹화: `기능별` 또는 `팀별`

---

### 🔹 4. 활용 방식

- 전략 페이지 하단에 `피드백 반영 히스토리` 섹션 삽입  
- 주간 회의 시 `상태별 요약 테이블`로 공유  
- 발표/보고서에 `반영 흐름 타임라인`으로 시각화  
- 회고 시 `미반영 항목` 중심으로 개선 논의
