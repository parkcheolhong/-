## 🚀 Web 프리뷰 데모 실제 업로드 흐름

### 1️⃣ GitHub 저장소 생성

- GitHub 접속 → 새 저장소 생성  
- 저장소 이름: `nadotong-preview`  
- 공개(Public) 설정  
- README.md 자동 생성 체크

---

### 2️⃣ 소스 파일 구성

```
nadotong-preview/
├── index.html
├── style.css
├── script.js
├── assets/
│   ├── images/
│   └── icons/
├── components/
│   ├── emotionCards.html
│   ├── challenge.html
│   ├── map.html
│   ├── interview.html
│   └── feed.html
```

> 각 기능별 HTML 파일은 `components/` 폴더에 분리  
> `index.html`에서 `iframe` 또는 `section`으로 연결

---

### 3️⃣ GitHub Pages 설정

- 저장소 → Settings → Pages  
- Source: `main` branch → `/root` 선택  
- 저장 후 자동 배포

---

### 4️⃣ 배포 링크 확인

- 배포 완료 후 링크 생성됨  
- 예시: `https://housing.github.io/nadotong-preview/`  
- Notion 전략 페이지, 제안서, 캠페인 키트에 삽입 가능

---

### 5️⃣ 테스트 및 수정

- 각 기능별 UI/UX 확인  
- 모바일/PC 반응형 체크  
- 오류 발생 시 `console.log`로 디버깅  
- 필요 시 GitHub에 커밋 → 자동 업데이트

---

## 🎯 활용 팁

- Web 프리뷰는 사용자 테스트, IR 발표, 캠페인 데모에 활용 가능  
- GitHub 저장소에 PDF/PPT, 이미지, 제안서도 함께 업로드 가능  
- Notion 전략 페이지에 링크 삽입 → 외부 공유 최적화
